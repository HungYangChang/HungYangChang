## Alignment Checklist

### For each project, verify:

- [ ] **Name match** — Portfolio project name matches GitHub repo display name
- [ ] **Description match** — One-sentence description is identical on both surfaces
- [ ] **Cross-links exist** — Portfolio links to GitHub repo; README links to portfolio/demo
- [ ] **Demo URL works** — Live demo link is functional and loads correctly
- [ ] **Screenshots current** — Screenshots in README match current state of live app
- [ ] **Stack listed** — Tech stack is consistent between portfolio and README

### Global checks:

- [ ] GitHub profile README links to portfolio
- [ ] Portfolio "About" section links to GitHub
- [ ] No stale/broken links on either surface
- [ ] Project ordering is consistent (strongest first on both)

## Naming Convention

Use kebab-case for GitHub repo names that match the portfolio project name:
- "Puppy Step" → `puppy-step`
- "Subscribe Manager" → `subscribe-manager`
- "Canada Count Down" → `canada-count-down`
- "Coach Book IQ" → `coach-book-iq`

## Action Items

1. Rename any misnamed GitHub repos to match convention
2. Update GitHub repo descriptions to match portfolio one-liners
3. Set homepage URL on each GitHub repo to the portfolio project page
4. Add portfolio link in each repo's README
5. Remove any dead links from both surfaces

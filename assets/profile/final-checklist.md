# Final Quality Review Checklist

Run this checklist before considering the GitHub presence "complete."

---

## 1. Profile README — 30-Second Recruiter Scan Test

- [ ] **Name and role** are visible without scrolling
- [ ] **Top 3 impact metrics** are in the first viewport (9×, 99.9%, 95%+)
- [ ] **Current employer/role** is mentioned
- [ ] **Contact links** (email, LinkedIn, portfolio) are easy to find
- [ ] **No typos or broken markdown** in the profile README
- [ ] Renders correctly on mobile (test via GitHub mobile app or narrow browser window)

## 2. Pinned Repositories

- [ ] **4–6 repos** are pinned
- [ ] Pin order follows: AI/LLM → Backend → Research → Product
- [ ] Each pinned repo has a **clear one-line description**
- [ ] Each pinned repo has **relevant topics/tags**
- [ ] Each pinned repo has a **homepage URL** (demo or portfolio link)

## 3. Flagship Repo Quality

For each pinned repo, verify:

- [ ] README follows the standard template
- [ ] Problem → Solution → Architecture → Stack → Setup → Demo → Results sections exist
- [ ] "Known Limitations" section is present (shows honesty and maturity)
- [ ] License file exists
- [ ] No credentials, API keys, or sensitive data in repo history
- [ ] Git history is clean (no "fix typo" × 20 commits in succession)

## 4. CI & Quality Signals

- [ ] At least one flagship repo has a green CI pipeline
- [ ] CI badge appears in the README for repos with pipelines
- [ ] Repos without CI have an explicit note explaining why

## 5. Collaboration Hygiene

- [ ] Issue templates are in `.github/ISSUE_TEMPLATE/`
- [ ] PR template is in `.github/PULL_REQUEST_TEMPLATE.md`
- [ ] CONTRIBUTING.md exists for repos accepting contributions
- [ ] SECURITY.md exists with responsible disclosure policy

## 6. Cross-Surface Consistency

- [ ] GitHub ↔ Portfolio project names match
- [ ] GitHub ↔ Portfolio descriptions match
- [ ] All cross-links are bidirectional and functional
- [ ] LinkedIn "Projects" section links to correct GitHub repos

## 7. External Link Validation

Test every link in the profile README:

- [ ] Portfolio URL loads
- [ ] LinkedIn URL loads
- [ ] Google Scholar URL loads
- [ ] Email `mailto:` link works
- [ ] Each featured project link works

## 8. Final Polish

- [ ] GitHub bio (short text under avatar) matches headline
- [ ] GitHub avatar is professional
- [ ] GitHub location is set (Montreal, Canada)
- [ ] GitHub "Available for hire" is toggled appropriately
- [ ] Profile README uses consistent formatting (headers, spacing, alignment)

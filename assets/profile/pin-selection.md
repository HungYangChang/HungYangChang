# Pinned Repository Selection

## Selection Strategy

Pin order follows **strongest technical signal first**, balancing AI/backend/product breadth.

Target: 4–6 repositories that demonstrate:
1. AI/LLM engineering depth (primary signal)
2. Production backend engineering
3. Research credibility
4. Full-stack product-building ability

## Recommended Pin Order

### Pin 1: LLM / AI Project (Strongest Signal)
**Positioning:** _"Multi-agent LLM orchestration system"_

Select the repo that best demonstrates function-calling pipelines, agent coordination, or RAG systems. If no dedicated public repo exists, create one showcasing a clean orchestration pattern.

### Pin 2: Backend / Infrastructure Project
**Positioning:** _"Production gRPC microservices with async orchestration"_

Select a repo showing system design, API architecture, or service reliability patterns.

### Pin 3: Research / ML Project
**Positioning:** _"High-throughput edge inference via NAS + pipeline parallelism"_

The PipeBERT or related research code — connects academic credibility to practical optimization.

### Pin 4: Mobile / Product App
**Positioning:** _"Full-stack pet companion app with real-time tracking"_

Puppy Step, Subscribe Manager, or Coach Book IQ — demonstrates shipping consumer products.

### Pin 5 (Optional): Open-Source Contribution or Tool
**Positioning:** _"Lightweight ML utility / developer tool"_

A focused OSS contribution or utility that shows community engagement.

### Pin 6 (Optional): Profile Repository
**Positioning:** _"This profile README"_

The `HungYangChang/HungYangChang` profile repo itself, if it demonstrates strong documentation and automation.
